# Reality Kernel — Rust CORE port (semantic preservation under pressure)

> **Status: VERIFIED on real silicon (Windows), scoped.** Authored with no toolchain in-session
> (`declared`), then compiled and run by the developer (`verified`): `cargo test` → 1 differential +
> 9 adversarial, 10/10 green. What that proves: the Rust CORE reproduces the Python oracle's

```bash
cd experiments/reality_kernel/core_rs

```

## What this port is — and is not

It is **not** "make the kernel fast." It is a proof that the four kernel objects survive what Python
could not exercise, **without losing the distinctions that created them**. The Python `reality_kernel`
is the reference model; this crate must match it.

```
Python RealityKernel  ──(differential tests, golden_kernel.tsv)──►  Rust CORE Runtime
```

The framing matters: "prototype → faster implementation" invites optimization drift; "reference model
→ substrate that must preserve the contract" forbids it. **Python defines what must be true; Rust
defines what must remain true under pressure.**

## The typed split (the physics of `compress ≠ sever`)

```
CommitPath   never drops · mutates canonical state · provenance required   (Core::apply)
ResolvePath  may defer / batch / drop · inspection only · cannot advance state   (ResolveRing — no apply)
```

Invariant, enforced by type, not by policy: **`dropped observation = allowed; dropped transition =
forbidden`.**

## Tests = the spec

`tests/differential.rs` builds the same scenario as the Python kernel and asserts every
`existence / diagnosis / resolution_path` against `golden_kernel.tsv` (generated by the Python oracle).
The five rows: gravity→present, relation_R→absent/severance, relation_S→unresolved/assumption_limit,
relation_T→unresolved/resource_limit, relation_NEVER_ASKED→unaccounted.

`tests/adversarial.rs` is the new thing Rust adds — **stress cannot collapse categories, and the
runtime fails loudly**:

```
artifact without provenance / event without source / receipt without digest   → refused at construction
severed prerequisite                          → refused, and state does NOT advance
buffer exhaustion                             → ResolvePath drops (counted); CommitPath never loses a transition
corruption of a digest                        → unresolvable (severed), never reconstructed by guess
eviction                                      → lineage is canonical, not a deletable cache; resolve still succeeds
many producers → one commit authority         → no dropped, no duplicated transition
producer panic mid-write                       → the incomplete transition is never visible
```

## Honest bounds and what is deferred

This crate is `std`-only (offline build) and uses `DefaultHasher` for digests — digests need only be
present and resolvable; the differential is over existence/diagnosis/resolution, never digest values.
**Deferred (named, not built):** the digest→lineage memory-scale benchmark (1e4 / 1e6 / 1e8 artifacts:
latency distribution, cache behaviour, footprint) — the failure case to watch is *digest exists,
lineage gone*; the frame-pipeline timing (`read → simulate → emit receipt → render digest → resolve
async`, where the frame never waits for explanation); and the Windows sub-granularity timing question
(full-frame spin vs. raising OS timer resolution). Those belong on real hardware with real scale.

The ceiling is unchanged: this is an **accounting system for state and ignorance**, not Reality. It
records what changed, why we think it changed, and what cannot currently be recovered — it does not
certify the model metaphysically correct. That boundary is what lets it scale without becoming
self-certifying.
