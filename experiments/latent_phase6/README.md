# Phase 6 — the inference contract (no edge without its price)

Phases 3–5 attached provenance to the **object** being learned (edge → support, representation → claim). Phase 6
moves the constraint to the **act of inference**: an estimator cannot produce a result without declaring what
made it possible. The atomic output is not "an edge with an accuracy" — it is an edge bound to the **price paid
to identify it**.

```
Before:  claim → attach floor ;  edge → attach support ;  representation → attach provenance
Now:     inference → attach price
```

The inversion: accuracy answers *"does this fit the regime I have?"*; it does **not** answer *"what alternative
worlds did I rule out?"* Different axes — so `accuracy ≠ identifiability`. A model can be 99% accurate and
weakly identified, because the 99% may hold only inside a narrow assumption set. The contract makes the **cost
mandatory** and the **accuracy optional and secondary** — and accuracy is not part of the output's identity.

This is the discipline layer, built **before** any estimator (the project's pattern: make the thing that
prevents self-deception exist before the thing tempted to self-deceive). A real estimator later either
satisfies the type or fails against it.

## Run

```bash
python3 experiments/latent_phase6/run.py     # stdlib only; deterministic
```

## The objects

- `IdentificationCost` — a **structured ledger, never a scalar**: `interventions`, `assumptions`,
  `domain_restrictions`, `unverified_dependencies`. (A cheap assumption and a rare intervention are not on one
  axis; collapsing them rebuilds the one-dimensional confidence object the project keeps rejecting.)
- `EstimatorOutput(edge, identification_cost, confidence_domain, accuracy=None)` — refuses construction with an
  empty cost. Identity = `edge + cost + domain`; `accuracy` is excluded from identity and can never substitute.
- `InferenceBudget` — composes edge prices into a graph-level ledger: intervention-purchased vs
  assumption-purchased, plus the composed cost ledger. A breakdown, not a score.

## Verified (the user's five tests + the ledger guard)

```
1. no free inference                       missing cost → construction failure
2. same edge, different price              do(g) vs assume(instrument_validity) → different identity
3. accuracy cannot upgrade provenance      99% accuracy + missing cost → still invalid; accuracy not required
4. domain is part of the claim             same edge+cost, different confidence_domain → different identity
5. cost composes                           graph budget = composition of edge provenance obligations
+  cost is a ledger, not a scalar          same count, different KIND of cost → distinguishable
```

## Honest bounds

This records the **declared** price; it does not verify the price was paid in valid coin — running the IV,
checking the invariance under intervention scarcity is the research-grade frontier (`do()` without a known
graph). The contract can sit empty-but-honest until a real estimator fills it. It would *earn* the separator
`accuracy ≠ identifiability`. `prediction ≠ inference identity`; `identity includes provenance`.

## The arc

```
P1 observation ≠ intervention      P2 causal relevance ≠ causal position
P3 edge ≠ edge-without-support      P4 latent ≠ discovered structure
P5 coordinate ≠ claim               P6 accuracy ≠ identifiability
                generated by:  identity includes provenance
```
